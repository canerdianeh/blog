It's been a while since I've had to do an actual AP-on-a-Stick (APoS) survey, but now that I'm going back out into the field to do this sort of thing, here's the step by step setup for setting up an Aruba AP for a survey. (And stay tuned for my semi-regular feature of what actually goes into my survey kit!)

Step 0: Hardware Selection

Ideally, you want to survey with the same model of AP/Antenna you plan on deploying - but that may not always be an option due to supply chain constraints, or a bunch of other factors. Just make sure that your survey makes a not of the differences in coverage between deployed and surveyed AP models, and make sure you do a validation survey after the fact.

As long-time readers are probably aware, I'm a stickler for good mounting solutions at Layer 0, and so whatever AP you get to survey, make sure that you have a way to mount it fairly cleanly. I'm a big fan of the Wi-Fi Stand and a Manfrotto lighting tripod for basic office surveys. For warehouses, things get a little bit trickier, because, while really big tripods that can reach 10-15m are available, they're a little unwieldy, and I prefer mounting the survey APs to structure using magnets.

You will also need some kind of battery that can provide sufficient PoE to the access point. I've recently acquired the MaxxWave survey battery which is very similar to existing 26400mAh USB batteries that I already had, but with different electronics. The MaxxWave battery appears to be identical in manufacture to the one that is being sold by Ventev, but for a significantly lower price. I had previously used the AccelTex battery, but the MaxxWave unit has the distinct advantage of being smaller, and able to charge/power over a Type C PD connection, which I already carry with me, so it reduces the size of my loadout.

From a setup standpoint, the specific AP model won't make much of a difference - the process is basically the same for any Aruba AP.

Step 1: Booting the AP in standalone mode

For the purposes of a survey, you'll want to set up the AP in standalone mode, which is a specialized version of Instant that suppresses and rejects controller discovery frames so that it doesn't attempt to join an existing Instant cluster or have other APs join it.

To boot an Aruba AP into Standalone mode, connect the requisite console cable to it and open your favorite terminal application (I'm a fan of ZOC) to the device, with 9600/N/8/1 and turn off all the extraneous RS232 stuff - don't need any handshaking, flow control, etc.

Note:

The AP-345, AP-303H, and all indoor 5XX series APs use the orange JY728A cable with a Micro-B connector on the AP side.

This is NOT a USB connection, (the USB to serial TTL interface is in the cable on the oversized Type A end) and regular Micro-B USB cables will not work.

Outdoor 560 and 570 series APs use a USB connection via Type C connector, and a Type A-Type C USB cable is included in the box with the AP.

Tip:

Most Aruba APs with integrated Bluetooth radios also have the ability to create a console connection over Bluetooth. If you find yourself without a console cable, this is how you can connect to an Aruba AP's console via Bluetooth

Now, set the environment to standalone and save the updated environment variables, and boot the AP. While you're in here, might as well go ahead and set up some of the other basics.

apboot>

setenv standalone_mode 1

setenv disable_prov_ssid 1

saveenv

Writing to UBI… done

boot ap

User:

Once you've gotten to the login prompt, wait a bit longer for the AP to finish doing its thing or you'll get a message saying the CLI isn't ready yet.

Log in using admin/admin or admin/serial # (if the first set doesn't work)

Step 2: Configuring the AP

Since we're already on the console, there's no reason not to configure the AP from here or mess with the WebUI. We disabled the provisioning SSID in the bootloader so we can start with a clean config.

Start by setting up the IP and removing the wired setmeUp config:

ip-address 172.18.19.20 255.255.255.0 172.18.19.1 1.1.1.1 arubanetworks.com

config terminal

enet0-port-profile default_wired_port_profile
enet1-port-profile default_wired_port_profile
no wired-port-profile wired-SetMeUp
no wlan access-rule wired-SetMeUp

Then create the SSIDs:

wlan access-rule Survey_5GHz
index 2
rule any any match any any any permit
exit

wlan access-rule Survey_2GHz
index 3
rule any any match any any any permit
exit

arm
wide-bands 5ghz
80mhz-support
band-steering-mode disable
air-time-fairness-mode default-access
client-aware

min-tx-power 9
exit

rf dot11g-radio-profile
max-tx-power 9
min-tx-power 9
disable-arm-wids-functions on
scanning-disable
no spectrum-monitor
exit

rf dot11a-radio-profile
max-tx-power 15
min-tx-power 15
disable-arm-wids-functions on

exit
